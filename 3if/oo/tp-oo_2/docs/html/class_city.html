Detailed Description

Manages the city's sensors and associated events.

Provides statistics based on the events which are added to it.

Constructor & Destructor Documentation

City::City ( )

Initializes the city to a default state.

A city can hold up to 20,000,000 events and 1,500 sensors; events must be added in chronological order and take place between May and September 2015.

Displays the optimal time to leave on a certain journey.

Given a set number of segments (corresponding to a sensor's area), this method finds the best time to leave on a journey comprised of those segments, in the order in which they were provided. To calculate the best time, it looks up the state of traffic at each segment at the time of arrival; a 'V' traffic state leads to a shorter journey than a 'J' state, for example.

Parameters
day7the day of the week (from 0 to 6)
hStartthe starting hour of the time range (from 0 to 23)

segCountthe number of segments
segmentsthe list of sensor ID's constituting the journey
